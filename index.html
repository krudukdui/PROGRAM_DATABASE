ระบบฐานข้อมูล

Database System — Concepts, Types & DBMS

1. ความหมายและองค์ประกอบ

💡

ฐานข้อมูลคืออะไร?

ฐานข้อมูล (Database) คือ การรวบรวมข้อมูลที่มีความสัมพันธ์กัน จัดเก็บอย่างเป็นระบบในคอมพิวเตอร์ เพื่อให้สามารถค้นหา เรียกใช้ และจัดการข้อมูลได้อย่างมีประสิทธิภาพ และลดความซ้ำซ้อนของข้อมูล

🌍 ตัวอย่างในชีวิตจริง
🏦ธนาคาร — ข้อมูลลูกค้า, บัญชี, ธุรกรรม
🏫โรงเรียน — ข้อมูลนักเรียน, ครู, วิชาเรียน
🏥โรงพยาบาล — ข้อมูลผู้ป่วย, แพทย์, ยา
🛒ร้านค้าออนไลน์ — สินค้า, คำสั่งซื้อ, ลูกค้า

🏗️ องค์ประกอบของระบบฐานข้อมูล

ระบบฐานข้อมูล (Database System) ประกอบด้วย 5 องค์ประกอบหลัก ที่ทำงานร่วมกัน

🔑 คำศัพท์สำคัญที่ต้องรู้

Table (ตาราง)
การจัดเก็บข้อมูลในรูปแบบแถวและคอลัมน์ เช่น ตารางข้อมูลนักเรียน
Field (ฟิลด์)
คอลัมน์ในตาราง แต่ละ Field เก็บข้อมูลประเภทเดียวกัน เช่น ชื่อ, อายุ
Record (ระเบียน)
แถวหนึ่งในตาราง แทนข้อมูลของสิ่งหนึ่งๆ เช่น ข้อมูลนักเรียน 1 คน
Primary Key (PK)
ฟิลด์ที่ระบุ Record ได้โดยไม่ซ้ำกัน และต้องไม่ว่างเปล่า เช่น รหัสนักเรียน
Foreign Key (FK)
ฟิลด์ที่อ้างอิง Primary Key ของตารางอื่น ใช้เชื่อมความสัมพันธ์ระหว่างตาราง
DBMS
Database Management System — ซอฟต์แวร์ที่ใช้จัดการฐานข้อมูล เช่น Microsoft Access
Query
การสอบถามหรือดึงข้อมูลจากฐานข้อมูล โดยระบุเงื่อนไขที่ต้องการ
Schema
โครงสร้างหรือแผนผังของฐานข้อมูล บอกว่ามีตาราง Field และความสัมพันธ์อะไรบ้าง

2. ชนิดของฐานข้อมูล

🌳 1. ฐานข้อมูลแบบลำดับชั้น (Hierarchical Database)

🌲

จัดเก็บข้อมูลในรูปแบบ ต้นไม้ (Tree Structure) มีความสัมพันธ์แบบ Parent–Child ข้อมูล Parent หนึ่งตัวมี Child ได้หลายตัว แต่ Child หนึ่งตัวมี Parent ได้เพียงตัวเดียว

✅ ค้นหาข้อมูลตามลำดับชั้นได้รวดเร็ว
❌ ยืดหยุ่นน้อย แก้โครงสร้างยาก
📌 ตัวอย่าง: IBM IMS, Windows Registry

🕸️ 2. ฐานข้อมูลแบบเครือข่าย (Network Database)

🕸️

พัฒนาจาก Hierarchical โดย Child หนึ่งตัวสามารถมี Parent ได้หลายตัว ทำให้โครงสร้างยืดหยุ่นกว่า แต่ซับซ้อนกว่า ความสัมพันธ์เป็น M:N

📦 สินค้า A
📦 สินค้า B
🏭 ผู้ผลิต 1
🏭 ผู้ผลิต 2
⟵ สินค้าหนึ่งชิ้นอาจมาจากหลายผู้ผลิต ⟶
✅ รองรับความสัมพันธ์ M:N ได้
❌ โครงสร้างซับซ้อน ยากต่อการดูแล
📌 ตัวอย่าง: IDMS, IDS

🗃️ 3. ฐานข้อมูลเชิงสัมพันธ์ (Relational Database) ⭐

🗃️

จัดเก็บข้อมูลในรูปแบบตาราง (Table/Relation) เชื่อมตารางด้วย Primary Key และ Foreign Key เป็นรูปแบบที่นิยมใช้มากที่สุดในปัจจุบัน ใช้ภาษา SQL ในการจัดการข้อมูล

🔷 4. ฐานข้อมูลเชิงวัตถุ (Object-Oriented Database)

🔷

จัดเก็บข้อมูลในรูปแบบวัตถุ (Object) ซึ่งรวมข้อมูลและพฤติกรรมไว้ด้วยกัน เหมาะกับงานที่ต้องการเก็บข้อมูลซับซ้อน เช่น รูปภาพ วิดีโอ หรือโปรแกรม OOP

Class: นักเรียน
📊 Attributes
- รหัสนักเรียน
- ชื่อ-นามสกุล
- คะแนน
⚙️ Methods
+ สมัครเรียน()
+ คำนวณเกรด()
✅ รองรับข้อมูลซับซ้อนได้ดี เช่น multimedia
❌ ซับซ้อน ยังไม่แพร่หลายเท่า RDBMS
📌 ตัวอย่าง: ObjectDB, db4o, Versant

📊 เปรียบเทียบฐานข้อมูลทั้ง 4 ชนิด

ชนิดโครงสร้างความยืดหยุ่นความซับซ้อนการใช้งานปัจจุบัน

3. คุณสมบัติของฐานข้อมูลที่ดี

⭐ ลักษณะและคุณสมบัติของฐานข้อมูลที่ดี

ฐานข้อมูลที่ออกแบบมาดี ควรมีคุณสมบัติสำคัญ 7 ประการ ดังนี้

01
🔄
ความซ้ำซ้อนน้อย
(Minimal Redundancy)
จัดเก็บข้อมูลแต่ละชุดในที่เดียว ไม่เก็บข้อมูลเดิมซ้ำหลายที่
❌ เก็บชื่อลูกค้าไว้ทั้งตาราง Orders และ Invoice
✅ เก็บชื่อลูกค้าในตาราง Customers ตารางเดียว
02
✅
ความถูกต้อง
(Data Integrity)
มีกลไกป้องกันข้อมูลผิดพลาด เช่น กำหนดชนิดข้อมูลและ Constraint
✅ กำหนด Data Type, Required Field, Validation Rule
03
🔗
ความสม่ำเสมอ
(Consistency)
ข้อมูลในทุกส่วนต้องสอดคล้องกัน ไม่ขัดแย้งกัน
✅ ใช้ Transaction เพื่อให้การเปลี่ยนแปลงสมบูรณ์ทุกครั้ง
04
🤝
การแชร์ข้อมูล
(Data Sharing)
ผู้ใช้หลายคนสามารถเข้าถึงข้อมูลพร้อมกันได้
✅ พนักงานหลายคนดูข้อมูลลูกค้าได้พร้อมกัน
05
🔒
ความปลอดภัย
(Security)
มีระบบกำหนดสิทธิ์การเข้าถึงข้อมูล ป้องกันผู้ไม่มีสิทธิ์
✅ กำหนด Username/Password และระดับสิทธิ์
06
🔍
ค้นหาง่าย
(Easy Retrieval)
สามารถค้นหาและดึงข้อมูลที่ต้องการได้รวดเร็ว
✅ ใช้ Index ในฟิลด์ที่ค้นหาบ่อย ทำให้ Query เร็วขึ้น
07
🔙
สำรองและกู้คืน
(Backup & Recovery)
มีระบบสำรองข้อมูลสม่ำเสมอ และสามารถกู้คืนได้เมื่อเกิดความผิดพลาด
✅ กำหนด Backup อัตโนมัติทุกวัน และทดสอบการ Restore

4. DBMS และหน้าที่

⚙️

DBMS คืออะไร?

Database Management System (DBMS) คือซอฟต์แวร์ที่ทำหน้าที่เป็นตัวกลางระหว่างผู้ใช้งานกับฐานข้อมูล ช่วยให้สามารถสร้าง จัดเก็บ ดึงข้อมูล และจัดการฐานข้อมูลได้อย่างมีระบบ

📋 หน้าที่หลักของ DBMS

📝
Data Definition (DDL)
กำหนดโครงสร้างฐานข้อมูล สร้าง/แก้ไข/ลบ Table, Field, Constraint
CREATE TABLE, ALTER TABLE, DROP TABLE
✏️
Data Manipulation (DML)
จัดการข้อมูล เพิ่ม แก้ไข ลบ และดึงข้อมูลจากฐานข้อมูล
INSERT, UPDATE, DELETE, SELECT
🔒
Security Control
ควบคุมสิทธิ์การเข้าถึงข้อมูล กำหนดว่าใครสามารถทำอะไรกับข้อมูลได้บ้าง
GRANT, REVOKE, User Authentication
🔄
Concurrency Control
จัดการเมื่อผู้ใช้หลายคนเข้าถึงข้อมูลพร้อมกัน ป้องกันข้อมูลขัดแย้ง
Locking, Transaction Management
💾
Backup & Recovery
สำรองข้อมูลและกู้คืนข้อมูลเมื่อเกิดความผิดพลาด
Backup Files, Transaction Log, Rollback
📊
Data Integrity
รับรองความถูกต้องของข้อมูล ด้วยการกำหนด Constraint และ Validation Rule
Primary Key, Foreign Key, Check Constraint

5. ข้อดีและข้อเสียของการใช้ฐานข้อมูล

✅ ข้อดีของการใช้ฐานข้อมูล

📉
ลดความซ้ำซ้อนของข้อมูล (Reduced Data Redundancy)
ข้อมูลแต่ละชุดเก็บในที่เดียว ไม่มีการเก็บข้อมูลซ้ำๆ ในหลายไฟล์ ประหยัดพื้นที่จัดเก็บและลดความสับสน
🔗
ความสม่ำเสมอของข้อมูล (Data Consistency)
เมื่อแก้ไขข้อมูลในที่เดียว การเปลี่ยนแปลงจะสะท้อนทั่วทั้งระบบ ไม่มีข้อมูลขัดแย้งกัน
🤝
แชร์ข้อมูลได้ (Data Sharing)
ผู้ใช้หลายคนในองค์กรสามารถเข้าถึงและใช้ข้อมูลร่วมกันได้พร้อมกัน
🔒
ความปลอดภัยของข้อมูล (Data Security)
สามารถกำหนดสิทธิ์การเข้าถึงข้อมูลได้ละเอียด ป้องกันการเข้าถึงโดยไม่ได้รับอนุญาต
🛡️
ความถูกต้องของข้อมูล (Data Integrity)
มีกลไกตรวจสอบความถูกต้องของข้อมูลอัตโนมัติ ด้วย Constraint และ Validation Rule
💾
มีระบบ Backup & Recovery
สามารถสำรองข้อมูลและกู้คืนได้เมื่อเกิดปัญหา ลดความเสี่ยงสูญหายของข้อมูล

❌ ข้อเสียของการใช้ฐานข้อมูล

💰
ต้นทุนสูง (High Cost)
ค่าใช้จ่ายในการซื้อ DBMS, ฮาร์ดแวร์ และการพัฒนาระบบค่อนข้างสูง
🧑‍🔧
ต้องการผู้เชี่ยวชาญ (Requires Expertise)
การบริหารจัดการฐานข้อมูลต้องใช้ผู้ที่มีความรู้และทักษะเฉพาะทาง เช่น DBA
⚠️
ความซับซ้อนสูง (High Complexity)
ระบบฐานข้อมูลมีความซับซ้อน ต้องใช้เวลาในการออกแบบและพัฒนา
🔌
ความเสี่ยงเมื่อระบบล้มเหลว (Single Point of Failure)
หากระบบ DBMS หยุดทำงาน ผู้ใช้ทุกคนจะไม่สามารถเข้าถึงข้อมูลได้

⚖️ เปรียบเทียบ: ระบบไฟล์ VS ระบบฐานข้อมูล

📄 ระบบไฟล์ปกติ
(Flat File System)
❌ ข้อมูลซ้ำซ้อนสูง
❌ ขาดมาตรฐาน
❌ ค้นหายาก ใช้เวลานาน
❌ ผู้ใช้หลายคนพร้อมกันไม่ได้
❌ ไม่มีระบบความปลอดภัย
❌ ไม่มี Backup อัตโนมัติ
✅ ต้นทุนต่ำ ใช้งานง่าย
✅ ไม่ต้องการผู้เชี่ยวชาญ
VS
🗄️ ระบบฐานข้อมูล
(Database System)
✅ ลดความซ้ำซ้อน
✅ มีมาตรฐาน สม่ำเสมอ
✅ ค้นหาได้รวดเร็ว
✅ รองรับผู้ใช้พร้อมกันได้
✅ มีระบบความปลอดภัย
✅ มี Backup & Recovery
❌ ต้นทุนสูงกว่า
❌ ต้องการผู้เชี่ยวชาญ
🎯

ผลการเรียนรู้: นักเรียนสามารถอธิบายความหมาย องค์ประกอบ ชนิด คุณสมบัติ และข้อดีข้อเสียของระบบฐานข้อมูล รวมถึงหน้าที่ของ DBMS ได้

📐 E-R Diagram (Entity-Relationship Diagram)

แผนภาพแสดงความสัมพันธ์ระหว่างข้อมูล ใช้สัญลักษณ์มาตรฐาน

Entity
สี่เหลี่ยม = Entity
สิ่งที่ต้องการจัดเก็บ
Relation
ข้าวหลามตัด = Relationship
ความสัมพันธ์ระหว่าง Entity
Attribute
วงรี = Attribute
คุณสมบัติของ Entity
PK
วงรีขีดเส้นใต้ = Primary Key
ค่าไม่ซ้ำ
ตัวอย่าง ERD: ระบบนักเรียนและวิชาเรียน

🔢 Normalization — การทำให้เป็นบรรทัดฐาน

กระบวนการปรับโครงสร้างตารางให้ถูกต้อง ลดความซ้ำซ้อนของข้อมูล

1NF
First Normal Form
กฎ: ทุก Field ต้องมีค่าเดียว (Atomic Value) และมี Primary Key
❌ Subject: "คณิต, วิทย์, อังกฤษ" (หลายค่า)
✅ แยกเป็นหลาย Record แทน
2NF
Second Normal Form
กฎ: ต้องอยู่ใน 1NF แล้ว และทุก Non-Key Field ต้องขึ้นกับ Primary Key ทั้งหมด
❌ ไม่มี Partial Dependency
✅ แยกตารางที่ขึ้นกับ Key บางส่วนออกไป
3NF
Third Normal Form
กฎ: ต้องอยู่ใน 2NF แล้ว และไม่มี Transitive Dependency
❌ Field A → Field B → Field C (ห้ามมี)
✅ ทุก Non-Key ขึ้นกับ PK เท่านั้น
🎯

ผลการเรียนรู้: นักเรียนสามารถออกแบบฐานข้อมูลและสร้าง E-R Diagram เพื่อวางแผนโครงสร้างข้อมูลได้

Primary Key

ฟิลด์ที่เป็นตัวระบุ Record ในตาราง ค่าไม่ซ้ำ ไม่ว่างเปล่า

🛡️

Referential Integrity คืออะไร?

กฎที่ควบคุมความสัมพันธ์ระหว่างตาราง เพื่อให้ข้อมูลมีความถูกต้องสม่ำเสมอ

  • ไม่สามารถเพิ่ม Record ใน Child Table ถ้าไม่มี Record ใน Parent Table
  • ไม่สามารถลบ Record ใน Parent Table ถ้ามี Record ที่อ้างอิงอยู่ใน Child Table
🎯

ผลการเรียนรู้: นักเรียนสามารถสร้างความสัมพันธ์ระหว่างตารางในฐานข้อมูลได้อย่างถูกต้อง

1. การจัดการผ่าน Form

📋

ทำไมต้องใช้ Form?

Form (แบบฟอร์ม) คือ หน้าต่างสำหรับรับและแสดงข้อมูล ที่ออกแบบมาให้ผู้ใช้ทำงานได้ง่าย สวยงาม และปลอดภัย แทนการเปิดตารางตรงๆ ซึ่งอาจเกิดข้อผิดพลาดได้

❌ เปิด Table ตรงๆ (Datasheet)
  • ดูยาก ไม่สวยงาม
  • เสี่ยงแก้ไข Field ผิด
  • ไม่มีการตรวจสอบข้อมูล
  • ผู้ใช้เห็นโครงสร้างตาราง
  • ไม่เหมาะกับผู้ใช้ทั่วไป
✅ ใช้ Form จัดการข้อมูล
  • หน้าตาสวยงาม เข้าใจง่าย
  • มีการตรวจสอบข้อมูลก่อนบันทึก
  • ซ่อนโครงสร้างตารางจากผู้ใช้
  • ใช้งานได้ปุ่มเดียว
  • เหมาะกับระบบงานจริง

Select Query

SELECT * FROM tblStudents WHERE Score > 70
✏️
Update Query
แก้ไขข้อมูลทีละหลาย Record พร้อมกัน
UPDATE tblStudents SET Class = 'ปวช.2' WHERE Class = 'ปวช.1'
🗑️
Delete Query
ลบข้อมูลทีละหลาย Record ตามเงื่อนไข
DELETE FROM tblStudents WHERE IsActive = False
➕
Append Query

INSERT INTO tblArchive SELECT * FROM tblStudents WHERE Year = 2567

Reports & Macros — Business Application

📊 แบบรายงาน (Reports)

📄

รายงานคืออะไร?

แบบรายงาน (Report) คือ การนำเสนอข้อมูลจากฐานข้อมูลในรูปแบบที่พร้อมพิมพ์หรือแสดงผล มักใช้สรุปข้อมูล สร้างใบเสร็จ หรือรายงานสถิติต่างๆ

ส่วนประกอบของ Report

Report Header ชื่อรายงาน, โลโก้ — แสดงครั้งเดียวที่หน้าแรก
Page Header หัวตาราง — แสดงทุกหน้า
Detail แถวข้อมูล — วนซ้ำสำหรับทุก Record
Page Footer หมายเลขหน้า — แสดงทุกหน้า
Report Footer สรุปรวม — แสดงครั้งเดียวท้ายรายงาน

⚡ แมโคร (Macro)

🤖

แมโครคืออะไร?

Macro คือชุดคำสั่งที่บันทึกไว้ล่วงหน้า เพื่อให้ทำงานซ้ำๆ โดยอัตโนมัติ ช่วยลดขั้นตอนการทำงาน เพิ่มประสิทธิภาพ

📋 Macro Actions ที่ใช้บ่อย

OpenForm
เปิด Form ที่กำหนด
OpenReport
เปิดหรือพิมพ์รายงาน
CloseWindow
ปิดหน้าต่างปัจจุบัน
GoToRecord
ไปยัง Record ที่กำหนด
RunQuery
รัน Query ที่กำหนด
MessageBox
แสดงข้อความแจ้งเตือน
FindRecord
ค้นหา Record ตามเงื่อนไข
Quit
ออกจากโปรแกรม Access

🛠️ วิธีสร้าง Macro

1

ไปที่แท็บ Create → Macro

2

เลือก Action จาก Dropdown ที่ต้องการ

3

กำหนด Arguments (พารามิเตอร์) ของ Action นั้นๆ

4

บันทึก Macro แล้วตั้งชื่อ เช่น mcrOpenStudentForm

5

นำ Macro ไปผูกกับปุ่มใน Form เพื่อใช้งาน
